Each line in the template file is one template.
Macro %x[row, col] refer to a specify token in the input data. row specifies the relative position from the current focusing token and col specifies the absolution position of the column.
/*unigram feature templates*/
U00:%x[-2,0]
U01:%x[-1,0]
U02:%x[0,0]
U03:%x[1,0]
U04:%x[2,0]

For each U0X:%x[row, col], CRF++ generates a set of feature functions.
The number of feature functions generated by a template amounts to (L*N), L is the number of output classes and N is the number of unique string expanded from the given template.
For example, for U0X:%x[row, 0], N is the number of given unique word; for U0Y:%x[row, 1], N represents how many features. 

unigram: [output tag]x[all possible strings expanded with a macro]

In my template, I check the previous 2 tokens before the current token, current token and post 2 tokens relative to current token.

/*bigram feature template*/
B

Use one bigram template ('B') means only combinations of previous output token and current token are used as bigram features.

A combination of the current output token and previous output token (bigram) is automatically generated. 
Generate a total of (L*L*N) distinct features, L is number of output classes, N is the unique features generated by the template.
bigram: [output tag]x[output tag]x[all possible strings expanded with a macro]
